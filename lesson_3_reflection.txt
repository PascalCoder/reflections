When would you want to use a remote repository rather than keeping all your work
local?

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    Because you might to have some files on remote but in the local repository.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    Forks is used to directly copy a GitHub repository into another.
	Clones can be used to copy a local repo into another one or a remote repo into
	a local one.
	Branches are used to make copy of other branches.

What is the benefit of having a copy of the last known state of the remote
stored locally?

    Remote branches can be accessed and modified by someone else. So having
	a local copy of that branch helps see changes that have been made if the
	remote and the copy in the local repo do not match.
